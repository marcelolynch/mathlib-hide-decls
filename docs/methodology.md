# Methodology

How candidates are selected, bundled, and scored. The dashboard's
Methodology tab renders the same content; this file is the source of
truth.

## 1. Census fields

`scripts/decl_census/DeclCensus.lean` runs in the elaborator and
emits one JSONL row per declaration in mathlib. The downstream
pipeline consumes these fields:

| field | meaning |
|---|---|
| `fq_name` | fully-qualified name |
| `defining_module` | module the decl was declared in |
| `kind` | `def` / `theorem` / `lemma` / `abbrev` / `inductive` / `structure` / … |
| `is_private` | already private in the source |
| `has_docstring` | doc comment present |
| `name_pattern` | classifier flag: `normal`, `underscore_prefix`, `simps_projection`, `auto_derived`, `aux`, `internal_namespace` |
| `forbidden_attrs` | attribute tokens from `policy.toml`'s blocklists that appear on the decl |
| `n_external_users` | references from other modules |
| `n_intra_module_refs` | references from the same module |
| `n_signature_refs` | same-module references occurring in the type signature, not the body |
| `signature_referenced_by_intra` | the actual list of decls that produced `n_signature_refs` |
| `meta_consumers` | extension registries that contain the decl by name (currently populated for `@[simps]`) |
| `simps_parent` | for `Foo.Simps.bar` decls, the parent struct's fq name |

## 2. Policy

Each decl passes through `pipeline/policy.toml`'s `hard_blocks`
before becoming a candidate.

| layer | blocks | reason |
|---|---|---|
| `forbidden_attrs` | `@[reducible]`, `@[implicit_reducible]`, `@[deprecated]`, `@[inline]` | Semantic attributes whose effect depends on visibility. |
| `build_rejected_attrs` | `@[simp]`, `@[norm_cast]`, `@[ext]`, `@[macro]`, `@[fun_prop]`, `@[positivity]`, `@[norm_num]`, `@[simps]`, `@[instance]`, … | Extensions that look up the decl by name at attribute elaboration. Lean rejects `private` on a decl carrying any of these. |
| `forbidden_name_patterns` | `simps_projection` (decls under `Foo.Simps.*`) | Generated by `@[simps]` on the parent struct, consumed by name without an attribute on the projection. |
| `forbidden_module_prefixes` | `Mathlib.Tactic.`, `Mathlib.Meta.`, `Mathlib.Lean.` | Elaborator-helper directories. Candidates from these are rejected at apply or reverted at build. Pre-filtering drops ~14% of candidate churn. |

A soft block additionally applies to theorems and lemmas: a theorem
with a docstring and `name_pattern=normal` is treated as public API
regardless of external-user count. The intent gate prefers false
negatives (under-flagging) over false positives (incorrectly hiding a
public theorem).

## 3. Tiering

| tier | membership | unit of action |
|---|---|---|
| **Tier 1** | `n_external_users == 0`, `n_signature_refs == 0`, intent-safe, policy-clean. Body refs in the same module are allowed (private decls remain visible inside the defining module, so proofs continue to type-check). | One `private` edit per decl. |
| **Tier 2** | Modules with ≥ 3 Tier-1 decls (grouped by `defining_module`). | One PR per module. |
| **Tier 3** | `kind ∈ {def, abbrev}`, `n_signature_refs ≥ 5`, `n_external_users ≤ 30`. | Move the hub and its in-signature dependents into a sub-namespace that the parent imports privately. |

## 4. Scoring

Per-module weight, used to rank tier-2 bundles:

```
score(m) = (1 − exp(−bcp / 100)) × (3 if T30 else 1) × (1 + log(1+edits) / 3)
```

| term | definition | source |
|---|---|---|
| `bcp` | Blast-cone critical-path seconds: the CP of the induced subgraph over all transitive importers of `m`. Per-module property of the build graph; independent of commit content. | `pipeline/p07_churn_blast.py` (upstream) |
| `T30` | 1 if `m` is in the 30 highest-leverage modules in the 1,500-commit window, else 0. Leverage = `edits × bcp`. | `data/churn_blast.json:top_leverage` |
| `edits` | Commits in the 1,500-commit window that touched `m`'s source. | `git log` walk over `master` |

Per-tier scoring:

- **Tier 1**: each decl inherits its module's `score(m)`.
- **Tier 2**: bundle score is `score(m)` weighted by `n_decls`.
- **Tier 3**: `bcp × n_signature_refs`.

## 5. T30: top-30 leverage list

Computed once per pipeline run from a walk of 1,500 commits of
mathlib master, attributing each commit's blast-cone CP to the
modules it touched, sorted by aggregate leverage. Membership shifts
as the import graph churns; the dashboard re-reads the list on every
regen.

## 6. Verification

Each retained privatization is verified by `lake build`. Two pipelines:

- **Bulk** (`pipeline/src/bulk_apply.py` + `pipeline/scripts/bulk_iterate.sh`):
  apply every tier-1 candidate in one commit, then loop
  build → revert breaking decls → rebuild until convergence. Output:
  one diff, full-mathlib verified.
- **Per-bundle** (`pipeline/scripts/iterate_bundle.sh`): apply per
  module with a 4-importer build cap. Output: a series of small
  commits.

Both pipelines end with a full-mathlib build.

The apply/revert primitives are unit-tested under `pipeline/tests/`:

- `test_policy_attrs.py` — 16 fixtures for the same-line and
  preceding-line attribute scanner in `policy.parse_attrs`.
- `test_revert_parser.py` — 12 fixtures for the build-log error
  parser in `bulk_revert.parse_errors`.

## 7. Calibration against PR 38702

[`pr-38702-trace.md`](pr-38702-trace.md) traces the policy + tiering
behaviour against mathlib4 PR
[#38702](https://github.com/leanprover-community/mathlib4/pull/38702)
(`chore(Data/Real): encapsulate real numbers`). The PR privatizes 40
decls organised around 4 hubs. The pipeline surfaces 1 hub
(`Real.mk`) as Tier 3, reaching 12 of the 40 decls through its
Co-located column. The other 28 are blocked by either the theorem-
intent gate or a hub-level filter (`@[reducible]`, `kind=ctor`,
or the `n_sig_refs < 5` threshold). The trace identifies four
concrete refinements that would raise coverage from 12 to 35+.
